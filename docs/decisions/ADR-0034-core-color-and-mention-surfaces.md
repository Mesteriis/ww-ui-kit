# ADR-0034 Core Color And Mention Surfaces

## Context

Color input and mention input are richer than plain text fields, but they still fit the baseline
field layer when their runtime stays bounded. The repo already has floating overlays, text inputs,
autocomplete behavior, and shared field wiring.

## Decision

`UiColorPicker` and `UiMention` land in `@ww/core`.

`UiColorPicker` contract:

- controlled-first `modelValue` string for the canonical color value
- shared overlay runtime for popup behavior
- saturation area, hue slider, optional alpha slider, text input, and preset swatches
- parsing is limited to hex and `rgba()` output generated by the component

`UiMention` contract:

- controlled-first text value
- explicit trigger characters and consumer-supplied suggestion records
- caret-anchored overlay list built on the governed floating path
- insertion formatting stays token-based and consumer-configurable

Async and data rules are explicit:

- color picker does not fetch palettes
- mention suggestions may be refreshed by consumers, but the component does not own fetching

Accessibility and keyboard rules are structural:

- color picker sliders stay keyboard accessible and expose color text inputs
- mention overlay uses combobox/listbox semantics with arrow navigation, `Enter`, `Escape`, and
  focus restore

## Consequences

- The repo gets baseline text-enrichment and color-input surfaces without creating a second systems
  package for bounded field behavior.
- Overlay, focus, and theming contracts stay shared with the existing field family.
- Apps keep ownership of async mention lookup and domain-specific token rules.

## Alternatives

- Treating both controls as systems would overstate their scope and duplicate the baseline field
  layer already in `@ww/core`.
- Shipping only composition recipes would duplicate caret handling, overlay behavior, and color
  parsing across consumers.

## Migration / Rollout

- Reuse core field styles and overlay motion.
- Keep mention insertion logic explicit and string-based rather than inventing a rich-text editor
  model.
- Keep color picker presets opt-in so the base surface stays small.
